## About Me

I’m currently doing my MS in Business Analytics at the University of Maryland. Before that, I studied Computer Science. Somewhere along the way, I realized I enjoy sitting at the intersection of business problems and technical structure.

## Where I’ve Worked

**Roche (Healthcare Analytics Intern)**  
I walked into a workflow where reports were manually stitched together every week.  
So I automated ~90% of it.

Built scalable SQL + Python pipelines, cleaned cross-platform data (Athena, Excel, SQL), and developed reporting structures that supported forecasting KPIs for drugs like Ocrevus and Hemlibra. The result: faster decisions, fewer errors, and 20+ hours saved per week.

**HorizonCore (Data Analyst Intern)**  
Built forecasting models hitting ~90% accuracy.  
Reduced reporting turnaround time by 60%.  
Improved inventory planning by 15% and helped cut excess stock by 8%.

Designed dashboards that managers actually used to shift budget toward high-performing channels. Not just pretty visuals — decisions.

## Things I’ve Built

**Pricing + Supply Chain Optimization Model**  
Created a 3-stage optimization pipeline (store pricing → warehouse allocation → vehicle routing).  
Modeled real constraints using AMPL + Gurobi.  
Simulated $58K+ in daily profit while keeping operations feasible.

This project made me love constraints.

**TransitPulse KPI System**  
Designed a relational database (9+ core tables) combining ridership, safety, and service data.  
Built SQL views to calculate YoY ridership change, reliability metrics, and crime-per-million-riders KPIs.  
Used it to identify multi-factor hotspot stations for better resource allocation.

Structured data > messy spreadsheets.
